## Key Design Decisions

1. **Thinking before execution** — The orchestrator enforces Intent → Plan → Reflect → Reason → Decide → Brief before any execution
2. **Ambiguity-first** — The intent engine specifically handles vague CEO messages ("Something feels wrong") by generating multiple hypotheses with confidence scores
3. **Bilingual** — All engines support both Vietnamese and English CEO messages
4. **Persistent memory** — Executive memory layer stores incidents, risks, priorities, and reasoning history to disk, surviving reboots
5. **Confidence honesty** — Reflection engine penalizes confidence for assumptions, alternatives, and weak evidence. Reports are never overconfident
6. **Dual-format briefs** — Every brief generates both WhatsApp-ready (concise) and Markdown (detailed) formats

## Processing Modes

| Mode | Trigger | Pipeline |
|------|---------|----------|
| quick | Status check, low confidence | Intent → Memory snapshot → Brief |
| full | Operational/tech concerns | Intent → Plan → Reflect → Brief |
| emergency | Urgent intervention | Intent → Fast findings → Emergency brief |
| strategic | Revenue/strategy/performance | Intent → Plan → Reflect → Business analysis → Decision matrix → Full brief |
