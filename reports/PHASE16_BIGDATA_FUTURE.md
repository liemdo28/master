# Phase 16 Big Data Future Layer

## Research Only
Do not deploy in this phase.

## Evaluation
- Apache Iceberg: useful for large analytical lakehouse tables.
- Trino: useful for federated SQL across lakehouse and operational stores.
- DataHub: useful for catalog and lineage.
- Superset/Metabase: useful for BI dashboards.
- TurboVec: monitor, but do not replace Qdrant.

## Decision
Keep Qdrant as Mi-Core vector store.
