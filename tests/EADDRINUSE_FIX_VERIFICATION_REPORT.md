## Fix Summary

Two changes applied to prevent the EADDRINUSE cascade:

| File | Change |
|------|--------|
| `mi-core/ecosystem.config.js` | Added `kill_timeout: 10000` (10s) to mi-core app block |
| `mi-core/server/src/index.ts` | Added `server.closeAllConnections?.()` before `server.close()` in `gracefulShutdown()` |

**Root cause addressed:** Keep-alive connections were holding the HTTP server open past PM2's default kill window, causing the old process to still own port 4001 when PM2 launched the new instance.
