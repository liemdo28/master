# Project DNA Generator

Part of the **Master Intelligence Layer**.

## Purpose

Reads `MASTER_INDEX.json` (generated by the master-indexer) and produces a `PROJECT_DNA.md` file in each project root directory, plus a summary table.

## What it generates

- **PROJECT_DNA.md** in each project root - contains identity, technology, business metadata, code metrics, dependencies, quality/health, and indexing info
- **PROJECT_DNA_SUMMARY.md** in this directory - a table summarizing all projects

## Prerequisites

- Node.js (no external dependencies required)
- `MASTER_INDEX.json` must exist at `../master-indexer/output/MASTER_INDEX.json`
- Run the master-indexer first if the index is missing

## Usage

```bash
node generate-dna.js
```

## Output

For each of the 30 projects in the index, a `PROJECT_DNA.md` is written to the project root with:

| Field | Default |
|-------|---------|
| Project Name | from index |
| Path | from index |
| Git Remote | from index |
| Branch | from index |
| Language | from index |
| Framework | from index |
| Purpose | UNKNOWN - NEEDS REVIEW |
| Business Function | UNKNOWN - NEEDS REVIEW |
| Owner | UNKNOWN - NEEDS REVIEW |
| Criticality | P2 (default) |
| Dependencies | from index |
| Known Risks | UNKNOWN - NEEDS REVIEW |
| Known Bugs | None recorded |
| QA Coverage | UNKNOWN - NEEDS REVIEW |
| Release Status | Active |
| Health Score | 75 - UNKNOWN |
| Last Indexed At | generation timestamp |

## Updating DNA files

Fields marked **UNKNOWN - NEEDS REVIEW** should be manually updated with actual project knowledge. Re-running the generator will overwrite all DNA files with fresh data from the index.

## Architecture

```
master-indexer/output/MASTER_INDEX.json
       |
       v
project-dna-generator/generate-dna.js
       |
       +---> [each project]/PROJECT_DNA.md
       +---> project-dna-generator/PROJECT_DNA_SUMMARY.md
```
