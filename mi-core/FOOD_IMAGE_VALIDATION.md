## Validation Framework

Every creative generated by the Restaurant Creative Engine V2 runs food validation before approval:

### 4 Required Checks

| Check | Description | Bakudan | Raw Sushi |
|-------|-------------|---------|-----------|
| sushi_visible | Sushi in photo (sushi/nigiri/roll/sashimi) | Optional | REQUIRED |
| ramen_visible | Ramen in photo (ramen/noodles/tonkotsu/spicy) | REQUIRED | Optional |
| food_quality_acceptable | Hero or feature quality asset present | REQUIRED | REQUIRED |
| restaurant_relevant | Restaurant food tag in any asset | REQUIRED | REQUIRED |

### Rejection Criteria

Creatives are **REJECTED** if:
- No hero/feature quality food photo found
- No restaurant-relevant food visible in assets
- Brand=raw_sushi AND sushi not visible in any asset
- Brand=bakudan AND ramen/noodles not visible in any asset
- Only generic graphics (no food photo tags)
- Only abstract designs (no food_quality_acceptable)
- Only placeholder cards (no real photo file)

## Validation Code

```typescript
// server/src/coo-v4/agents/restaurant-creative-engine.ts
export function validateFoodAssets(assets: FoodAsset[], brand: Brand): FoodValidationResult {
  const tags = assets.flatMap(a => a.tags);

  const sushi_visible = brand === 'raw_sushi'
    ? tags.some(t => ['sushi', 'nigiri', 'roll', 'sashimi'].includes(t))
    : true;

  const ramen_visible = brand === 'bakudan'
    ? tags.some(t => ['ramen', 'noodles', 'tonkotsu', 'spicy'].includes(t))
    : true;

  const food_quality_acceptable = assets.some(a => a.quality === 'hero' || a.quality === 'feature');
  const restaurant_relevant = tags.some(t =>
    ['ramen','sushi','hero','restaurant','noodles','nigiri','roll','chef','premium'].includes(t)
  );

  // REJECT if any check fails
  let reject_reason: string | undefined;
  if (!food_quality_acceptable)           reject_reason = 'No hero/feature quality food photo found';
  if (!restaurant_relevant)               reject_reason = 'No restaurant-relevant food visible';
  if (brand === 'raw_sushi' && !sushi_visible) reject_reason = 'Sushi not visible in assets';
  if (brand === 'bakudan' && !ramen_visible)   reject_reason = 'Ramen not visible in assets';

  return { sushi_visible, ramen_visible, food_quality_acceptable, restaurant_relevant,
           reject_reason, passed: !reject_reason };
}
```

## Generic / Placeholder Detection

The engine flags and rejects these patterns:
- `qualite = 'placeholder'` — any file with "placeholder", "generic", "card" in name
- Files < 20KB (too small to be real photography)
- HTML-only files (old V1 flyer format)
- No food tags in asset library
